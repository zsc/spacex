## 7.1 概述：重新定义载人飞船

当 SpaceX 在 2006 年赢得 NASA 的商业轨道运输服务（COTS）合同时，整个航天界都在观望这家成立仅 4 年的公司如何挑战已有 50 年历史的载人航天工业。传统的载人飞船设计——从水星号到航天飞机——都遵循着一套根深蒂固的设计范式：一次性使用、极度保守的冗余设计、天价的"航天级"组件。

### 传统设计惯性的枷锁

```
传统载人飞船设计理念
┌─────────────────────────────────────────────┐
│  阿波罗/联盟模式             航天飞机模式    │
├─────────────────────────────────────────────┤
│  • 一次性使用               • 部分可重用      │
│  • 降落伞溅落/着陆          • 滑翔着陆        │
│  • 烧蚀式热防护             • 瓷砖热防护      │
│  • 全硬件冗余               • 五重冗余计算机   │
│  • 成本：$1B+/次            • 成本：$450M/次  │
└─────────────────────────────────────────────┘
```

这些设计选择并非基于物理学的第一性原理，而是基于历史惯性、风险厌恶和既得利益。SpaceX 看到了彻底重新思考的机会。

### 第一性原理的革命性应用

埃隆·马斯克和 SpaceX 团队问了一个根本性问题："如果我们从零开始设计一个 21 世纪的载人飞船，它应该是什么样子？"答案形成了 Dragon 的核心设计理念：

1. **可重复使用性优先**：不是"能否重用"，而是"为什么不能重用 100 次？"
2. **软件定义的冗余**：用现代计算能力替代复杂的硬件备份
3. **商用组件的航天化**：不是购买昂贵的"航天级"部件，而是让商用部件满足航天需求
4. **垂直整合制造**：控制供应链就是控制成本和质量
5. **快速迭代**：失败快、学习快、改进快

## 7.2 Dragon 1：商业货运的破冰之旅

### COTS 计划：NASA 的范式转变

2006 年，NASA 做出了一个历史性决定：不再完全依赖传统的成本加成合同模式，而是通过商业轨道运输服务（COTS）计划，以固定价格合同激励私营公司开发货运能力。这个决定的背景是航天飞机即将退役，而国际空间站需要可靠的补给线。
